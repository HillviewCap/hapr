HAPR-PROC-002 — Runs as Non-Root User

HAPR-TLS-001 — Minimum TLS Version 1.2

High Severity (6 findings)
HAPR-PROC-001 — Chroot Enabled

Add a chroot directive to the global section pointing to an empty, dedicated directory. Example: chroot /var/lib/haproxy

HAPR-ACL-002 — Admin Paths Restricted

Use ACLs to restrict admin paths to trusted source IPs or require HTTP basic authentication. Example: acl admin_path path_beg /admin /manager acl trusted_src src 10.0.0.0/8 http-request deny if admin_path !trusted_src

HAPR-ACL-003 — Rate Limiting Configured

Implement rate limiting using stick tables with http_req_rate or conn_rate tracking. Example: stick-table type ip size 100k expire 30s store http_req_rate(10s) http-request deny deny_status 429 if { sc_http_req_rate(0) gt 100 }

HAPR-ACL-005 — Stats Access Restricted

Restrict stats access with authentication and IP filtering. Example: stats auth admin:securepassword stats admin if LOCALHOST acl stats_allowed src 10.0.0.0/8 http-request deny if !stats_allowed

HAPR-LOG-003 — Stats Page Secured with Authentication

Enable authentication on the stats page and bind it to a restricted address. Example: stats enable stats uri /haproxy-stats stats auth admin:strong-password-here stats hide-version

HAPR-SCAN-003 — Valid Certificate Chain

Ensure the certificate file includes the full chain in order: server certificate, intermediate certificates, and optionally the root CA. Use openssl s_client -connect host:port to verify the chain.
